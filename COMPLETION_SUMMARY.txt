KEY DESIGN DECISIONS
================================================================================

1. Generic BrainInterface (Local Definition)
   Problem: Cross-package imports → rootDir constraint violation
   Solution: Define interface locally in adapter with generic types
   Result: Framework has zero imports from game-specific packages

2. Framework Owns Orchestration, Adapter Owns Communication
   Problem: Adapter was becoming bloated with logic
   Solution: Framework provides GameLoop, adapter only implements GameSession
   Result: Clean separation, maximum reusability

3. Readonly/Mutable Type Separation
   Problem: BrainStats fields readonly, cannot assign during calculation
   Solution: Mutable internals, readonly public API
   Result: Type-safe immutability guarantee

4. Framework-Agnostic UI Components
   Problem: Components need to work in React, Vue, vanilla JS
   Solution: Pure formatting functions with zero dependencies
   Result: Zero framework imports in ui-components.ts
